Case study

The pipeline this page ships through

Walk the full pipeline, stage by stage →

git push ──▶ GitHub Actions ──▶ docker build ──▶ smoke test (curl)
                    │
                    └──▶ terraform fmt + validate ──▶ aws s3 sync ──▶ S3 static website

A real lesson: curl: (56)

The first CI run failed with curl: (56) Recv failure. The cause: docker run -d only means the container started — Nginx could still be initializing. A race condition.

The fix: readiness thinking

The pipeline now retries the smoke test while the service warms up, prints logs when something fails, and always cleans up its containers. Small platform, real reliability lesson.

This very page ships through the pipeline it documents — validated in CI, then synced to S3 — see the repository.
